Java-八股文-1-Java语言基础问题

Posted on 2022-04-28 In Java Views:

#Java语言基础问题

1.解释下什么是面向对象？面向对象和面向过程的区别？

面向过程是把完成功能的过程分成若干个步骤， 将这些步骤分散到不同的函数
也可以说是方法中去，接下来只要依次调用这些步骤代表的方法即可。
面向对象则是将整个需求按照特点、功能等将整个系统划分为若干个类(最常见的也就是模拟现实世界进行设计)，，
将系统分解为不同的对象以及这些对象的行为。

2.解释一下面向对象的三大特性

封装:
封装指的是把客观的事物进行抽象包装成类，并且把自己的属性和方法包装起来，根据具体情况
来决定是否对外隐藏或开放
继承:
继承指的是，继承某类的子类可以使用父类的所有功能，并且还可以在此基础上对父类的功能进行拓展
多态:
多态指的是，某些方法对于不同的类具体的实现可能不同，但是他们可以通过相同的方法进行调用

3.JDK JRE JVM

JDK是Java的核心，包括了JRE，一些java工具以及java类库
JRE是java的运行环境，包含JVM标准实现以及java类库
JVM是java虚拟机，是运行Java进程的虚拟计算机

4.Java创建对象的方法

1.new关键字
2.反射：Class.newInstance
3.反射：Constructor.newInstance
4.实现cloneable接口调用clone
5.反序列化

5.接口与抽象类的区别

1.接口只允许定义方法，抽象类允许实现方法
2.实现接口用implements关键字，并且一个类允许实现多个接口。
继承抽象类用extends关键字，一个类只允许继承一个类

6.HashCode相同，equal()的结果就是true吗？

两个对象equals相等，则它们的hashcode必须相等，反之则不一定。
两个对象==相等，则其hashcode一定相等，反之不一定成立。

7.Java方法参数是传值还是传引用？

传值，基本类型传值的拷贝。引用类型传引用的地址的拷贝。

8.如何实现对象的克隆？

浅拷贝:当对象被复制时，只复制对象本身和其中包含的值类型的成员变量，而引用类型的成员对象并没有复制。
深拷贝:将原型对象的所有引用对象也复制一份给克隆对象。
1.浅拷贝：实现cloneable接口，调用clone方法
2.深拷贝：序列化反序列化的方式

9.泛型是如何工作的？

Java在编译的时候会进行类型擦除，也就是说Java在编译的时候，代码中的泛型类型已经是确定的，泛型是被擦除掉的。

10.限定通配符与非限定通配符

非限定通配符：
<T> 任意类型
限定通配符：
<? T extends F>必须是F本身或是继承F的子类
<? T super F>必须是F本身或是F的父类

11.String,StringBuffer,StringBuilder

String是final的，线程不安全
StringBuffer是可变的，线程安全
StringBuilder是可变的，线程不安全。
字符串拼接速度：String<StringBuffer<StringBuilder
首先String是final的，因此如果要拼接就需要重新new一个String对象，重新赋值
其次StringBuffer为了线程安全进行了一些加锁的操作，牺牲了一定的效率

12.StringBuilder为什么线程不安全？StringBuffer为什么线程安全？

StringBuilder的append方法没有考虑线程同步的问题，但是在StringBuffer中
用synchronize关键字进行了线程同步。

13.字节流（InputStream,OutputStream)、字符流(Reader,Writer)

字节流中最小的数据单元是字节，字符流是字符

14.BIO、NIO、AIO

BIO:
BIO指阻塞IO，在从IO未获取到数据的情况下就会一直等待，等待会使当前线程阻塞
Non-Blocking IO:
NIO指非阻塞式IO,无论什么情况直接获取数据，获取失败则一直重试。
IO复用：
IO复用是指单个线程同时处理多个IO操作，通过一些操作轮询当前可以处理的IO操作
select:线性轮询符合条件的操作
epoll:记住当前条件下对应的IO操作进行调用
