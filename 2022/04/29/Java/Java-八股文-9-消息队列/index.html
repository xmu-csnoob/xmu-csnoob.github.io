1.消息队列的基本作用？

①解耦
服务端只负责消息的产生，至于消息发给谁交给消息决定
②异步
可以将一个同步的步骤，通过发送消息的形式转变为异步
③削峰
大量请求涌入时，会把请求压入消息队列，让服务端按照自己的速度拉取处理
