#Java-八股文-7-计算机网络

1.为什么一定要三次握手？

建立TCP连接的前提是，双方都要确保各自的收发能力都是正常的。
客户端向服务器发请求时，服务器可以确定客户端发消息能力正常，
服务器回执时，客户端能够确定服务器收发能力都正常，
但是服务器并不确定客户端收消息能力是否正常，因此需要客户端再向服务器发消息，
以确保客户端收消息正常。

2.为什么一定要四次挥手

四次挥手是因为，在服务器响应可以关闭连接时，服务器有些数据可能还未发送完毕
因此当且仅当服务器数据发送完毕后，才会告诉服务器关闭连接。

3.TCP与UDP有哪些区别？各自应用场景？

4.HTTP1.0，1.1，2.0 的版本区别

HTTP 1.0:默认短连接，但支持长连接
HTTP 1.1：默认长连接
HTTP 2.0：复用TCP连接

5.HTTP如何实现长连接？

利用TCP的保活机制，保活机制的工作原理就是，通过在服务器端设置一个保活定时器，当定时器开始工作后就定时的向网络通信的另一端发出保活探测的TCP报文，如果接收到了ACK报文，那么就证明对方存活，可以继续保有连接；否则就证明网络存在故障。

6.输入URL到显示页面的全过程

1.DNS域名解析
2.建立TCP连接，发送HTTP请求
3.服务器处理请求，返回HTTP报文
4.浏览器根据报文渲染页面

7.Cookie和Session的区别

cookie和session都是用来跟踪浏览器用户身份的会话方式
(1)cookie数据存放在客户的浏览器上，session数据放在服务器上
(2)cookie不是很安全，别人可以分析存放在本地的COOKIE并进行COOKIE欺骗,如果主要考虑到安全应当使用session
(3)session会在一定时间内保存在服务器上。当访问增多，会比较占用你服务器的性能，如果主要考虑到减轻服务器性能方面，应当使用COOKIE
(4)单个cookie在客户端的限制是3K，就是说一个站点在客户端存放的COOKIE不能3K
(5)所以：将登陆信息等重要信息存放为SESSION;其他信息如果需要保留，可以放在COOKIE中

Posted on 2022-04-29 In Java

Java-八股文-6-MySQL

1.数据库的三范式是什么？

1NF:属性不可分
2NF:所有属性完全依赖于主键
3NF:属性不间接依赖于主键

2.MySQL索引是什么？

索引是对数据库表中一列或多列的值进行排序的一种结构。
MySQL索引的建立对于MySQL的高效运行是很重要的，索引可以大大提高MySQL的检索速度。

3.索引底层数据结构？

B树，B+树，Hash
B树
1.关键字分散在整棵树
2.根据任何一个关键字可以唯一的找到一个节点
3.搜索可能在非叶子节点结束
4.树高每多一层，就多一次磁盘I/O消耗

B+树
1.关键字只出现在叶子节点中，并且这些节点用链表相连接

Hash
哈希索引就是采用一定的哈希算法，把键值转为哈希值，检索时只需要进行一次哈希算法。
但哈希索引不支持范围查询

4.索引分类

逻辑分类

按功能划分
1.主键索引：将表的主键自动设置为索引
2.唯一索引：要求索引列唯一且不为null
3.普通索引
按列数划分
1.单例索引：一个索引只包含一个列，一个表可以有多个单例索引。
2.组合索引：一个组合索引包含两个或两个以上的列。查询的时候遵循 mysql 组合索引的 “最左前缀”原则，即使用 where 时条件要按照建立索引的时候字段的排列方式放置索引才会生效。

物理分类

1.聚簇索引(InnoDB)
聚簇索引（clustered index）不是单独的一种索引类型，而是一种数据存储方式。
这种存储方式是依靠B+树来实现的，根据表的主键构造一棵B+树且B+树叶子节点存放的都是表的行记录数据时，
方可称该主键索引为聚簇索引。聚簇索引也可理解为将数据存储与索引放到了一块，找到索引也就找到了数据。

2.非聚簇索引(MyISAM)
非聚簇索引：数据和索引是分开的，B+树叶子节点存放的不是数据表的行记录。

5.聚簇索引的优缺点

优点：
1.数据访问更快，因为聚簇索引将索引和数据保存在同一个B+树中，因此从聚簇索引中获取数据比非聚簇索引更快
2.聚簇索引对于主键的排序查找和范围查找速度非常快
缺点：
1.插入速度严重依赖于插入顺序，按照主键的顺序插入是最快的方式，否则将会出现页分裂，严重影响性能。因此，对于InnoDB表，我们一般都会定义一个自增的ID列为主键（主键列不要选没有意义的自增列，选经常查询的条件列才好，不然无法体现其主键索引性能）
2.更新主键的代价很高，因为将会导致被更新的行移动。因此，对于InnoDB表，我们一般定义主键为不可更新。

6.覆盖索引

覆盖索引包含了一个sql语句中所有条件出现的属性。

7.索引失效

1.where语句带有or，部分条件无索引
2.like查询以%开头
3.where进行字符串查询，不带引号
4.索引列参与计算
5.条件中只有组合索引中的部分属性

8.主从复制原理


主要包含三个线程：
1.master（binlog dump thread) 主要负责Master库中有数据更新的时候，会按照binlog格式，将更新的事件类型写入到主库的binlog文件
2.slave (I/O thread) 该线程用于请求Master，Master会返回binlog的名称以及当前数据更新的位置、binlog文件位置的副本
3.slave (sql thread) 当Slave检测到中继日志有更新，就会将更新的内容同步到Slave数据库中，这样就保证了主从的数据的同步

9.主从同步策略

「同步策略」：Master会等待所有的Slave都回应后才会提交，这个主从的同步的性能会严重的影响。
「半同步策略」：Master至少会等待一个Slave回应后提交。
「异步策略」：Master不用等待Slave回应就可以提交。
「延迟策略」：Slave要落后于Master指定的时间。

10.MVCC

MVCC是MySQL乐观锁实现的一种方式，是行级锁。
通过两个对用户不可见的版本号属性来进行操作。

1.解释下什么是面向对象？面向对象和面向过程的区别？

面向过程是把完成功能的过程分成若干个步骤， 将这些步骤分散到不同的函数
也可以说是方法中去，接下来只要依次调用这些步骤代表的方法即可。
面向对象则是将整个需求按照特点、功能等将整个系统划分为若干个类(最常见的也就是模拟现实世界进行设计)，，
将系统分解为不同的对象以及这些对象的行为。

2.解释一下面向对象的三大特性

封装:
封装指的是把客观的事物进行抽象包装成类，并且把自己的属性和方法包装起来，根据具体情况
来决定是否对外隐藏或开放
继承:
继承指的是，继承某类的子类可以使用父类的所有功能，并且还可以在此基础上对父类的功能进行拓展
多态:
多态指的是，某些方法对于不同的类具体的实现可能不同，但是他们可以通过相同的方法进行调用

4.Error和Exception的区别？

Error是系统级别的错误，是程序员无法改变和处理的，是编译时出现的错误。

5.运行时异常与受检异常有何异同？

运行时异常，表示程序代码在运行时发生的异常，程序代码设计的合理，这类异常不会发生
受检异常跟程序运行的上下文环境有关，即使程序设计无误，仍然可能因使用的问题而引发

6.throw 和 throws 的区别？

不能。首先线程实现了runnable接口的run方法，run方法没有抛出任何的异常，这也就说明线程内的方法
只能在线程内部自己处理。

Java容器

1.ArrayList和LinkedList的区别？

ArrayList底层是通过动态数组实现，LinkedList通过链表实现。
ArrayList根据index访问更快，LinkedList需要遍历
ArrayList新增元素效率较低，涉及到新建数组与元素复制，LinkedList可以直接增加节点。
LinkedList不支持RandomAccess

2.ArrayList 实现 RandomAccess 接口有何作用？为何 LinkedList 却没实现这个接口？

实现RandomAccess接口能让继承List的类支持随机快速访问。
实现了这个接口的类在遍历时使用for，没有实现则使用迭代器模式
哪个快用哪个

3.ArrayList 的扩容机制？

元素数量达到容量时触发扩容，扩容为原来的1.5倍

4. Array 和 ArrayList 有何区别？什么时候更适合用 Array？

Array可以定义为基本类型，ArrayList只能定义为对象类型。
Array只能存放同一类的对象，ArrayList可以存放同一类（或接口）及其子类

5.HashMap 的实现原理/底层数据结构？JDK1.7 和 JDK1.8

JDK 1.7：
HashMap=数组+单链表
put操作：
key为null单独处理
不为null则计算哈希值——>找到对应的index——>遍历index对应的链表——>插入数据
resize操作：
定义新的Entry，新容量——>将原HashMap的各节点映射到新的Entry中去
get操作：
key为null单独处理
不为null则根据key直接取值

JDK 1.8:
HashMap=数组+链表+红黑树
put操作：
计算哈希值找index——>判断index节点类型——>按照链表/红黑树的方法插入——>链表长度=8转红黑树

6.HashMap 多线程死循环问题？

HashMap是采用链表解决Hash冲突，因为是链表结构，那么就很容易形成闭合的链路，这样在循环的时候只要有线程对这个HashMap进行get操作就会产生死循环。
在单线程情况下，只有一个线程对HashMap的数据结构进行操作，是不可能产生闭合的回路的。
那就只有在多线程并发的情况下才会出现这种情况，那就是在put操作的时候，如果size>initialCapacity*loadFactor，那么这时候HashMap就会进行rehash操作，随之HashMap的结构就会发生翻天覆地的变化。很有可能就是在两个线程在这个时候同时触发了rehash操作，产生了闭合的回路。

7.HashMap 与 ConcurrentHashMap 的区别是什么?

HashMap不支持并发，ConcurrentHashMap是线程安全的。
ConcurrentHashMap底层是由若干个Segment构成的，对于这些segment有上锁

8.HashSet 的实现原理？

HashSet实现Set接口，基于HashMap实现，hashMap的key不重复，作为hashMapKey的hashSet元素自然不重复

9.LinkedHashMap是如何实现的

继承了HashMap，但采用双向链表来作为底层实现，每个节点多出来的指针就可以去记录插入的顺序。

1. 并发和并行的区别?

并发，在操作系统中，是指一个时间段中有几个程序都处于已启动运行到运行完毕之间，且这几个程序都是在同一个处理机上运行
并行，在操作系统中，是指同一时间多个程序都在运行，真正的并行只能在多核处理器上实现

2.线程和进程的区别

1.进程是操作系统分配资源的基本单位，线程是使用资源的基本单位，也是程序运行的基本单位
2.进程切换上下文更慢，耗费更多资源。线程切换上下文很快，耗费资源较少。
3.进程之间通信是需要一些额外手段的，线程之间则可以通过访问进程中线程共享的资源来传递信息。

3.什么是守护线程？

守护线程是程序运行的时候在后台提供一种通用服务的线程。

4.Java开启线程的方式？

1.继承Thread类
thread类本质上实现了runnable接口，其start方法内调用了runnable的run方法
2.实现Runnable接口
实现Runnable接口后将其作为参数初始化Thread，调用start方法
3.实现Callable接口

5.Runnable 和 Callable 有什么区别？

Runnable的run方法返回值是null，Callable的call方法有返回值，并且支持泛型
Runnable的run方法只可以抛出运行时异常，不能捕获异常，Callable的call方法支持抛出异常

10.ThreadLocal变量

ThreadLocal 类用来提供线程内部的局部变量。这种变量在多线程环境下访问（通过get 和 set 方法访问）时能保证各个线程的变量相对独立于其他线程内的变量。ThreadLocal 实例通常来说都是 private static 类型的，用于关联线程和线程上下文。
可以通过 ThreadLocal 在同一线程

11.谈谈 synchronized 和 ReenTrantLock 的区别？

synchronized是java关键字，是java原生锁，无需手动上锁与释放锁，执行完后会自动释放。
ReenTrantLock则是一个实现类，需要调用lock与unlock对资源上锁与解锁。

12.volatile和synchronized

1.volatile本质上是告诉进程该变量的值要从内存中读取，synchronized则是对变量上锁
2.volatile只能对基本类型使用，synchronized适用于类
3.volatile不能保证原子性，因此可能会造成脏读的情况。sync可以保证一致性

13. 说下对 ReentrantReadWriteLock 的理解？

读写锁维护一个读锁和一个写锁，读锁是共享的，写锁是排他的。也就是说同一时间可能有多个
线程持有读锁，但只可能有一个线程持有写锁

14.说下对悲观锁和乐观锁的理解？

悲观锁是指，在尝试对数据进行修改时，先对数据进行加锁，再去修改。
乐观锁是指，默认数据不会发生冲突，只有在提交的时候才去对数据冲突进行检测。

15.乐观锁实现方式

1.用CAS修改数据，没有冲突则提交——>会导致ABA问题
2.给数据附加一个版本号，每次修改数据时更新版本号，根据版本号判断是否有冲突

17.什么是线程池？为什么引入线程池？

线程池是指在初始化一个多线程应用程序过程中创建一个线程集合，然后在需要执行新的任务时重用这些线程而不是新建一个线程。
线程池能够降低系统资源消耗（重复申请销毁线程），也可以帮助程序管理线程。

Java-八股文-1-Java JVM

1.说一下 JVM 的主要组成部分？及其作用？

①类加载器：加载类文件到内存
②栈区：栈也叫栈内存，是java程序的运行区，是在线程创建时创建，它的生命周期跟随线程的生命周期，线程结束栈内存释放；
③堆内存：类的加载器读取了类文件之后，需要把类、方法、常变量放到堆内存中
④方法区：所有的类级数据将存储在这里，包括静态变量。每个JVM只有一个方法区，它是一个共享资源;

2.谈谈对运行时数据区的理解？

主要是五部分：程序计数器、方法区、虚拟机栈、本地方法栈、堆
![image](source/pictures/Java/Java-八股文-1-Java JVM/1.png)

3. 堆和栈的区别是什么？

栈中存储的是局部变量
存储的是数组和对象（其实数组就是对象），凡是new建立的都是在堆中，堆中存放的都是实体（对象），实体用于封装数据，而且是封装多个（实体的多个属性），如果一个数据消失，这个实体也没有消失，还可以用，所以堆是不会随时释放的，但是栈不一样，栈里存放的都是单个变量，变量被释放了，那就没有了。

4.判断垃圾可以回收的方法

①引用计数法
的基本思想是堆中每个对象都有一个引用计数器，每当有一个地方引用它，计数器的值就加1；当引用失效时，计数器的值就减1。计数器值为0的对象可以被当做垃圾回收。
②可达性分析法
Java语言是通过可达性分析算法来判断对象是否存活的。这个算法的基本思想是以 GC Root 对象为起点，开始往下搜索，所走过的路径称为引用链，当 GC Root 和一个对象之间没有任何引用链相连时，则证明此对象是不可用的。

5.垃圾回收算法

①标记-清除算法
②标记-复制算法
③标记-整理算法

6.分代收集算法？为什么要使用分代收集算法？

根据对象的存活周期，把内存分成多个区域，不同区域使用不同的回收算法回收对象。
新生代：由于新生代对象的生命周期一般较短，因此可以用标记-复制算法来进行收集
老年代：由于老年代对象存活率比较高，可以用标记-清除算法或标记-整理算法来进行收集。

8.浮动垃圾

用户线程和GC线程是并行运作的，如果说在一次GC中，从GC Root的位置开始走到某处，
在已经走过的路径上，用户线程又产生了垃圾，这样的垃圾就是浮动垃圾。

9.堆内存碎片

在Java堆中，若干个对象有可能是连续存储的。这个时候如果触发了GC，导致一些对象被回收，
则会在这些对象原有的位置留下内存的空洞，这些空洞有可能不能让任何一个新的对象容纳进去，就会产生内存空间上的
浪费，也就是内存碎片。
Java在发现堆碎片化后会触发Full GC，让对象都靠一侧移动，以变成一个连续的存储空间。

10.常用的垃圾收集器有哪些？

①CMS
CMS 全称 Concurrent Mark Sweep 并发标记清除垃圾回收器。CMS 是一种以获取最短停顿时间为目的的垃圾回收器。提到停顿时间，
我们都知道任何垃圾回收器在进行工作的时候都会出现 STW，Stop the World 停止用户进程，这对业务来说只很难接受的，但是现在市面上所有的垃圾回收器都无法避免这个问题，只能最大化的去优化，从而降低停顿的时间。
②G1
G1 全称 Garbage-First 是一种面向服务器的垃圾回收器，通过将堆内存划分为多个 Region 来实现可预测的停顿时间模型。在 G1 当中，新生代和老年代已经不再是物理隔离，而都是被划分一个个 Region 区域。
正是由于这种可预测的时间停顿模型让 G1 成为了一个高吞吐量的垃圾回收器。G1 能充分利用 CPU，多核环境下可以缩短 STW 的时间。

11.常用的监控JVM的参数？

1.top 查看当前进程的使用情况(CPU占有率，内存使用情况，服务器负载情况)
2.jps 查看本机jvm进程
3.jinfo 可以输出或修改的java进程的一些参数
4.jstat 可以用来监视jvm内存内的各种堆和非堆的大小及其内存使用量
5.jstack：堆栈跟踪工具，一般用于查看某个进程包含线程的情况
6.jmap：打印出某个java进程（使用pid）内存内的所有对象的情况。一般用于查看内存占用情况。
7.jconsole：一个java GUI监视工具，可以以图表化的形式显示各种数据。并可通过远程连接监视远程的服务器的jvm进程。
